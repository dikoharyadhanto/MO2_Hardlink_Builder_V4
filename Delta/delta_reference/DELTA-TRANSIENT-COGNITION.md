# Transient Cognitive Exchange Specification

**Delta Ecosystem — Ephemeral Cognition Governance**
**Authority Tier:** Operational Doctrine (subordinate to the Delta Constitution and the operational governance protocol)
**Owner:** Director

> This document defines what constitutes ephemeral cognition within the Delta Ecosystem, how it flows between agents and sessions, and the rules governing its termination and non-persistence. This doctrine operationalizes the Transient Cognition principle established in the Delta Constitution. In any conflict between this document and the operational governance protocol or the Delta Constitution, those documents prevail.

---

## Section 1 — Definition of Transient Cognition

**Transient cognition** is any reasoning, inference, analysis, or conclusion generated by an agent during the active execution of a session that:

1. Was not formally approved and promoted to a governance artifact
2. Does not carry an assigned document code (e.g., STRAT, WO, IMPL)
3. Exists to serve the immediate reasoning needs of the current session only
4. Has not been written to the MCP memory graph with Director approval

Transient cognition is the cognitive substrate of agent work — the thinking that produces deliverables. It is not itself a deliverable.

**Transient cognition is not lesser or inferior.** It is the correct class for reasoning artifacts. Classifying it as ephemeral protects governance integrity by preventing cognitive residue from accumulating governance authority it was never assigned.

## Section 2 — Taxonomy of Transient Cognitive Artifacts

### 2.1 Cognitive Exchange

A **Cognitive Exchange** is any in-session conversational turn, analysis, or reasoning step produced by an agent while working toward a formal deliverable. Cognitive Exchanges are the most common form of transient cognition.

**Characteristics:**
- Exists within a single session context
- Not formally titled or assigned a document code
- May inform a formal deliverable but does not itself become one
- Decays at session boundary

**Examples:** An agent's reasoning about which approach to take in a STRAT, a mid-session hypothesis about a dependency relationship, a draft paragraph that was later revised.

### 2.2 Audit Output

An **Audit Output** is a conversational assessment, critique, or risk evaluation produced by GPT or PPX during a review cycle. Audit Outputs inform decisions but are not themselves governance artifacts.

**Characteristics:**
- Conversational in format — no formal document code assigned
- Informs the document being reviewed but does not amend it
- The document being reviewed retains sole authority over its domain
- Decays at session boundary unless formally captured in a CSO

**What makes an Audit Output permanent:** If the Director explicitly instructs that an audit finding be incorporated into a governance document (e.g., a STRAT revision, a protocol amendment), it becomes part of that document. The Audit Output itself does not gain permanence — only the document that absorbs it does.

### 2.3 Reasoning Artifact

A **Reasoning Artifact** is a structured but informal output produced during analysis — draft architectures, comparison tables, decision trees, alternative designs explored and discarded. These are produced as tools for arriving at a decision, not as decisions themselves.

**Characteristics:**
- May appear highly structured and document-like
- Does not carry governance authority regardless of structure
- The absence of a formal document code is the determining factor
- Decays at session boundary

### 2.4 Speculative Hypothesis

A **Speculative Hypothesis** is any conclusion, prediction, or assessment that has not been validated against governance documents or confirmed by the Director. Agents regularly produce these during planning and analysis.

**Characteristics:**
- May not be acted upon as if it were authoritative
- Must be validated before influencing a formal deliverable
- Decays immediately upon being superseded by a validated conclusion
- Never persisted to MCP memory

## Section 3 — Cognitive Flow Between Agents

Transient cognition does not flow directly between agents. It is produced within a session and terminates with that session. The mechanism for transferring relevant cognitive state between agents is the **CSO (Cognitive State Object)** — a formal document that captures approved session state at a point in time.

### 3.1 The CSO as the Cognitive Handoff Boundary

```
Agent A Session
    │
    ├── Transient Cognition (reasoning, drafts, hypotheses)
    │       │
    │       └── DECAYS at session end
    │
    └── CSO Section 9 — Memory & Persistence Candidates
            │
            └── Director reviews → approved items ONLY
                    │
                    ├── Written to MCP Memory (Operational or Constitutional tier)
                    └── Incorporated into formal documents
```

**What crosses the session boundary:** Only formally approved outputs — governance documents (STRAT, WO, IMPL, WALK), MCP memory entities approved via CSO, and the CSO itself as an immutable log.

**What does not cross the session boundary:** All transient cognition. An agent in a new session must not attempt to reconstruct prior session reasoning from memory. It must bootstrap from: MCP memory, the Registry, and formal governance documents.

### 3.2 No Implicit Cognitive Continuity

Agents must not assume cognitive continuity between sessions. The following are prohibited:

- Referencing prior session reasoning as if it carries current authority ("as I concluded last session...")
- Treating a prior Cognitive Exchange as an approved decision unless it was formally incorporated into a document
- Reconstructing discarded reasoning artifacts and treating them as active

When a new session begins, the agent's authority context is entirely derived from Layer 1–3 of the cognition stack (Runtime Metadata, MCP Memory, Registry) and from formal governance documents. Prior session reasoning has no standing.

### 3.3 Multi-Agent Cognitive Isolation

When multiple agents operate in sequence (e.g., GMN → ANT → CDC), each agent's transient cognition is isolated from the others. An ANT session does not inherit GMN's reasoning residue — it inherits only GMN's formal deliverable (STRAT) and any MCP memory entities that were approved post-session.

This isolation is a governance feature, not a limitation. It prevents reasoning errors, unchallenged assumptions, and cognitive drift from propagating downstream.

## Section 4 — Termination Rules

### 4.1 Natural Termination

Transient cognition terminates naturally at the session boundary. No explicit deletion or archival action is required. The session context ends; the cognitive residue ends with it.

### 4.2 Premature Termination

Transient cognition may be terminated before session end when:

- A hypothesis is disproven during the session — it is superseded and no longer active
- A draft is replaced by a revised version — the prior draft loses standing
- The Director redirects the session — prior reasoning toward the abandoned direction terminates

Superseded transient artifacts must not be resurrected within the same session unless the Director explicitly reverses direction.

### 4.3 Prohibited Persistence Paths

The following actions constitute illegal persistence of transient cognition and are governance violations:

| Prohibited Action | Why Prohibited |
|---|---|
| Writing session reasoning directly to MCP without CSO + Director approval | Bypasses the Ephemeral → Operational promotion gate |
| Including reasoning residue as observations in a formal governance document | Introduces unvalidated cognition into the authority layer |
| Carrying forward a discarded design alternative as if it were an active option | Resurrects terminated transient cognition |
| Treating an Audit Output as a governance decision without Director incorporation | Elevates transient cognition to authority it was not assigned |
| Referencing prior session reasoning in a new session as authoritative | Implies cognitive continuity that does not exist |

## Section 5 — Promotion Pathway

Transient cognition may be promoted to a persistent artifact only through the following pathway. There is no shortcut.

```
1. Agent identifies a candidate insight during session execution
        ↓
2. Agent records candidate in CSO Section 9 — Memory & Persistence Candidates
   (with: proposed tier, source context, justification)
        ↓
3. Director reviews CSO Section 9 at session close
        ↓
4a. Director approves → agent executes MCP write (Operational or Constitutional tier per DELTA-MEMORY-DOCTRINE.md)
4b. Director approves incorporation into formal doc → document is updated
4c. Director rejects → candidate decays with session; no write occurs
```

**A candidate not reviewed by the Director does not become persistent by default.** Inaction is rejection. If the Director does not review a CSO before the next session begins, the candidates in that CSO are treated as expired.
